Hoje vou compartilhar alguns "alias" de comandos Git que utilizo no meu dia-a-dia. Essa dica irá te ajudar a simplificar comandos longos que têm muitas opções, tornando-os mais curtos, mais fácil de memorizar e consequentemente seu dia mais produtivo e feliz.

Como definir e usar alias:

Para definir seus "alias" Git, eu sugiro alterar diretamente no arquivo de configuração Git:

$ vi ~/.gitconfig

Agora basta incluir seus "alias" nessa sessão. Para exemplificar vamos criar um "alias" para validar o status do repositório (git status)

[alias]
      s=status

Vamos validar o "alias" , vá o seu terminal e digite:

$ git s
On branch main
Your branch is up to date with 'origin/main'.

nothing to commit, working tree clean

Listando seus alias

Para listar todos os seus "alias" basta rodar o comando:

$ git config --global -l
user.name=f0rmig4
user.email=f0rmig4@mail.com
core.editor=vim

Minha lista de alias

Vou listar aqui meus "alias" , copie, cole, compartilhe =)

[alias]
         a=add
         s=status
         c=checkout
         l=checkout --
         b=checkout -b
         co=commit
         p=push
         m=merge
         unstage=reset HEAD --
         find = log --pretty=\"format:%Cgreen%H %Cblue%s\" --name-status --grep

Git alias é um recurso muito útil para otimizarmos comandos comuns e repetitivos. Git permite que você defina quantos "alias" for necessário. Mas cuidado, definir muitos "alias" pode dificultar memoriza-los.
