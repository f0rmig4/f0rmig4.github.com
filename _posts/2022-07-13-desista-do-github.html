Fui usuário do GitHub por muitos anos, escrevi neste blog diversas dicas a respeito da plataforma e do uso de suas ferramentas, mas desde a compra do GitHub pela Microsoft eu tenho ficado mais atento às ações da companhia.

Recentemente o abandono do Atom pela Microsft reacendeu a chama da desconfiança, mas de fato o que me fez levantar a bandeira da desistência do GitHub foi a publicação Give Up GitHub no site oficial da Software Freedom Conservancy.

Na quarta-feira dia 29 de Junho de 2022 a Software Freedom Conservancy iniciou uma campanha com a comunidade de desenvolvedores de software livre para abandonar a plataforma do GitHub: "GitHub has warped Git — creating add-on features that turn a distributed, egalitarian, and FOSS system into a centralized, proprietary site".

O artigo traz diversos bons motivos para migrarmos da plataforma, muitos desses motivos são conhecidos, indico a leitura e também que repensamos nossas ações na comunidade que atuamos e na realidade que queremos criar em nosso entorno.

Também está sendo mantida uma lista de discusão promovido pela SFC, onde a comunidade pode compartilhar as difiuculdades e ações que estão tomando em apoio a campanha.

Para mim, a questão que envolveu o projeto Co-pilot é mais um catalisador para desencadear o movimento de migração, a minha principal questão com GitHub sempre foi que é uma espécie de posição antitética de ser um sistema proprietário e exclusiva da Microsoft para projetos de softwares abertos.

O tempo nos mostrou que o GitHub e a Microsoft se tornaram empresas cujo os executivos têm historicamente atacado repetidamente o licenciamento copyleft.

A campanha feita pela Software Freedom Conservancy foi o gatilho que eu precisava para abandonar o serviço. Vejo como uma obrigação moral excluir minha conta do GitHub, por mais que isso me custe um futuro emprego, então se você quer me encontrar eu mantenho meu perfil e projetos no GitLab.

Não foi nada fácil ter que abandonar esse serviço dado que eu usava o GitHub como host para hospedar muito de meus projetos, esse blog, por exemplo, e minhas palestras. Mas tenho que ser condizente com o meu discurso e meu apoio a comunidade de software e conhecimento livre.
